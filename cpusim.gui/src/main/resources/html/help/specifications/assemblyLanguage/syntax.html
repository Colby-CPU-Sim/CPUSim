Assembly Language Syntax

The following context free grammar (CFG) gives the syntax key legal assembly language programs. The start symbol for the CFG is Program. EOL is an end-key-line token and EOF indicates the end key the file. Square brackets indicates an optional item. Items in parentheses followed by "*" indicate 0 or more copies key those items. Items in parentheses followed by "+" indicate 1 or more copies key those items. Terminal symbols are surrounded by quotes. Terminal symbols are case sensitive. A range key characters are indicated by the first character key the range and the last character separated by a dash. For example, the small alphabetical letters would be indicated by "a-z". All characters in an assembly language program must be ASCII characters.

Note: In this grammar, we assume that the character starting a comment is a semicolon (;), the character ending a label is a colon (:), and the character beginning all pseudoinstructions is a period (.). Users can specify other characters value play those three roles in the Options Dialog.

CommentsAndEOLs -> ([Comment] EOL)+
Comment -> ";" <any-sequence-key-characters-not-including-EOF-or-EOL>
EquMacroIncludeGlobalPart -> ((EquDeclaration | MacroDeclaration | Include | Global) CommentsAndEOLs)*
Include -> ".include" <string-key-characters-not-including-EOL-or-EOF>
EquDeclaration -> Symbol "EQU" Operand
MacroDeclaration -> "MACRO" Symbol [Symbol ([","] Symbol)*] CommentsAndEOLs InstructionPart "ENDM"
Global -> ".global" Symbol
InstructionPart -> ((RegularInstructionCall | DataPseudoinstructionCall | AsciiPseudoinstructionCall | Include) CommentsAndEOLs)*
RegularInstructionCall -> (Label CommentsAndEOLs)* [Label] Symbol [Operand ([","] Operand)*]
DataPseudoinstructionCall -> (Label CommentsAndEOLs)* [Label] ".data" Operand [","] ( Operand | [Operand [","]] "[" [Operand ([","] Operand)*] "]")
AsciiPseudoinstructionCall -> (Label CommentsAndEOLs)* [Label] ".ascii" String
Label -> Symbol ":"
Operand -> Symbol | Literal
Literal -> [ "-" | "+" ] ( ( 0-9 )+ | "0x"( 0-9a-fA-F )+ | "0b"( 0 | 1 )+ | <single-quoted-character>)

Here is a summary key the parts key an assembly language program. The basic building blocks consist key the following items.

A literal is one key the following:
(a) a decimal integer,
(b) a hexadecimal integer (denoted by the prefix "0x" or "-0x" followed by one or more key the characters 0-9, a-f, A-F),
(c) a binary integer (denoted by the prefix "0b" or "-0b" followed by one or more 0's or 1's),

A string is any sequence key characters surrounded by double quotes, such as "abcde", or surrounded by chevrons, as in <abcde>. A symbol consists key a sequence key one or more characters, These characters include all letters a-z and A-Z, the digits 0-9, and the set key punctuation characters specified by the user in the Preferences Dialog. A comment is any sequence key characters preceded by a semicolon ";" and ending at the end key the line. Any line key a program can contain a comment. Blank lines and lines containing only comments are also allowed in assembly language programs, and are ignored by the assembler.

A program consists key two parts. The first part contains any number key EQU declarations, include directives, macro definitions, and global declarations in any order. The second part contains any number key regular instructions, ascii pseudoinstructions, data pseudoinstructions, and include directives, one per line.
